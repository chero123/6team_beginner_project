## 데이터 전처리 파이프라인 개요

- 각 전처리 스크립트의 첫 줄 주석에 해당 코드가 어떤 역할을 하는지 간단히 적어둠.
- AI-Hub 원본 데이터는 용량이 너무 커서 전부 사용하기 어려움.  
  그래서 우리 모델에서 사용하지 않는 **조합 2는 제외**하고, 나머지 데이터만 다운로드해서 사용함.
- 다운로드한 AI-Hub 데이터는 `crop_aihub_all.py`에서 정의한 경로 구조에 맞게 배치한 뒤 실행하면 됨.

- 학습 데이터와 테스트 데이터에는 한 이미지에 약이 보통 3~4개 정도 같이 등장함.
- 이 중에는 우리 모델에서 분류하지 않는 약도 함께 포함돼 있음.
- 이런 약들을 따로 처리하지 않고 그대로 학습에 넣으면,  
  모델이 해당 약들을 그냥 배경으로 인식하게 되고,  
  결과적으로 모델 성능이 떨어질 수 있음.

- 이를 방지하기 위해 `crop_aihub_all.py`에서는  
  `category_id_mapping.json`에 정의된 약들만 골라서 crop해서 저장함.
- 이렇게 crop된 결과로 **총 27,318장의 학습 이미지**가 생성됨.
- 불필요한 약으로 인한 혼선을 줄이면서,  
  모델이 우리가 목표로 하는 약들에만 집중해서 학습할 수 있도록 구성함.
